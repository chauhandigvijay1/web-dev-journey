# Phase 4 - STL (C++)

A structured STL phase for **DSA + problem solving** in C++.

This phase is designed to make STL:

* Easy to revise
* Easy to practice
* Easy to use in coding rounds

Instead of learning STL randomly, this phase is organized **topic-wise** with:

* Notes
* Curated questions
* Revision notes
* Proper folder structure

---

## Overview

This phase focuses on learning the **Standard Template Library (STL)** in C++.

STL is one of the most important parts of C++ for:

* Coding interviews
* Online assessments
* DSA
* Competitive programming
* Fast implementation

### Main Goal

Build strong command over:

* STL syntax
* STL containers
* STL algorithms
* STL usage in DSA problems

---

## What This Phase Covers

This phase includes the most important STL topics you actually need for DSA:

1. Pair
2. Vector
3. Iterators
4. Algorithms
5. Stack
6. Queue
7. Priority Queue
8. Set
9. Map
10. Unordered Map
11. Unordered Set
12. Comparator / Custom Sorting

---

## Learning Goals

By the end of this phase, you should be able to:

* Choose the right STL container quickly
* Write STL syntax without confusion
* Use common STL algorithms confidently
* Solve frequency / lookup / heap / sorting-based problems
* Reduce code length and write cleaner solutions

## Why This Phase Matters

A lot of DSA becomes easier only after STL is clear.

STL helps in:

* Reducing implementation time
* Writing cleaner code
* Solving interview problems faster
* Avoiding unnecessary manual code

> If STL is weak, many later DSA topics feel harder than they actually are.
